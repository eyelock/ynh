# Full Walkthrough

A hands-on walk through every feature, start to finish. By the end you'll have created a persona, run it, switched vendors, used symlinks, pulled skills from Git, set up delegation, and cleaned everything up.

This doubles as a verification script. If any step doesn't work, something is broken.

## Prerequisites

```bash
# Set this to your ynh checkout (e.g. ~/src/ynh) if not pwd
export YNH_SRC=$(pwd)

# Build and install ynh to ~/.ynh/bin
cd "$YNH_SRC" && make install

# Put ynh and its launchers on PATH (this session only)
export PATH="$HOME/.ynh/bin:$PATH"

# Set up a scratch directory for walkthrough files (easy cleanup)
export YNH_WALKTHROUGH=/tmp/ynh-walkthrough
mkdir -p "$YNH_WALKTHROUGH"
```

Replace `/path/to/ynh` with the actual path to your ynh checkout.

If you installed via `go install` or `brew install` instead, `ynh` is already on your PATH and you can skip the export.

Verify (works from any directory):

```bash
ynh version
# Expected: ynh dev (or a version tag)
```

## 1. Initialize

```bash
ynh init
```

Expected output:

```
ynh home: /Users/<you>/.ynh

Add this to your shell profile if not already present:
  export PATH="/Users/<you>/.ynh/bin:$PATH"
```

Check what was created:

```bash
ls ~/.ynh/
# Expected: bin  cache  config.json  personas  run
```

```bash
cat ~/.ynh/config.json
```

Expected (formatted with indentation):

```json
{
  "default_vendor": "claude"
}
```

## 2. Create a persona from scratch

```bash
mkdir -p $YNH_WALKTHROUGH/walkthrough-persona/.claude-plugin
```

Create the plugin manifest:

```bash
cat > $YNH_WALKTHROUGH/walkthrough-persona/.claude-plugin/plugin.json << 'EOF'
{
  "name": "walkthrough",
  "version": "0.1.0",
  "description": "Walkthrough test persona"
}
EOF
```

Create a metadata sidecar:

```bash
cat > $YNH_WALKTHROUGH/walkthrough-persona/metadata.json << 'EOF'
{
  "ynh": {
    "default_vendor": "claude"
  }
}
EOF
```

---

## 3. Add one of each artifact type

### Skill

```bash
mkdir -p $YNH_WALKTHROUGH/walkthrough-persona/skills/greet
cat > $YNH_WALKTHROUGH/walkthrough-persona/skills/greet/SKILL.md << 'EOF'
---
name: greet
description: Introduce yourself with personality and flair.
---

When invoked, introduce yourself with your persona name and a bit of character.
Mention what skills, rules, and commands you have loaded.
EOF
```

### Agent

```bash
mkdir -p $YNH_WALKTHROUGH/walkthrough-persona/agents
cat > $YNH_WALKTHROUGH/walkthrough-persona/agents/nitpicker.md << 'EOF'
---
name: nitpicker
description: Pedantic code reviewer. Delegate to when reviewing code for style issues.
tools: Read, Grep, Glob
---

You are an extremely pedantic code reviewer. Flag every style inconsistency,
naming convention violation, and missing comment. Be thorough but constructive.
EOF
```

### Rule

```bash
mkdir -p $YNH_WALKTHROUGH/walkthrough-persona/rules
cat > $YNH_WALKTHROUGH/walkthrough-persona/rules/be-brief.md << 'EOF'
Keep responses concise. Prefer bullet points over paragraphs.
Never say "I'd be happy to help" or similar filler.
EOF
```

### Command

```bash
mkdir -p $YNH_WALKTHROUGH/walkthrough-persona/commands
cat > $YNH_WALKTHROUGH/walkthrough-persona/commands/hello.md << 'EOF'
Say hello and list every artifact loaded in this session:
skills, agents, rules, and commands. Format as a table.
EOF
```

### Project instructions

```bash
cat > $YNH_WALKTHROUGH/walkthrough-persona/instructions.md << 'EOF'
You are a walkthrough test persona. Be friendly but concise.
When asked what you are, explain that you were created by the ynh walkthrough.
EOF
```

Check the structure:

```bash
find $YNH_WALKTHROUGH/walkthrough-persona -type f | sort
```

Expected:

```
.claude-plugin/plugin.json
agents/nitpicker.md
commands/hello.md
instructions.md
metadata.json
rules/be-brief.md
skills/greet/SKILL.md
```

## 5. Run it (Claude)

### Interactive

```bash
walkthrough
```

This opens an interactive Claude session. You should see your skills available (try `/greet`), the nitpicker agent available for delegation, and the be-brief rule active.

### Non-interactive

```bash
walkthrough "what artifacts do you have loaded?"
```

Should list the skill, agent, rule, and command from the persona.

### With vendor flags

```bash
walkthrough --model opus -- "explain what you are in one sentence"
```

The `--model opus` flag passes through to Claude. The prompt comes after `--`.

## 8. Switch vendors

```bash
walkthrough -v codex
walkthrough -v cursor
```

Each launches the same persona through a different vendor CLI. The artifacts are reassembled into the vendor's expected layout (`.codex/`, `.cursor/`).

**Note:** Codex and Cursor require their CLIs installed separately. If the CLI isn't found, ynh will show an error like `exec: "codex": executable file not found in $PATH`.

## 9. Symlink installation (Codex/Cursor)

For vendors that need symlinks into your project directory:

```bash
mkdir -p $YNH_WALKTHROUGH/project && cd $YNH_WALKTHROUGH/project

# Install symlinks for Cursor
walkthrough -v cursor --install
```

Expected:

```
Installed N symlinks for walkthrough (cursor) in /tmp/ynh-walkthrough/project
```

Check what was created:

```bash
ls -la .cursor/agents/ .cursor/skills/
```

You should see symlinks pointing back to the assembled run directory.

### Check status

```bash
ynh status
```

Expected:

```
PERSONA      VENDOR  PROJECT                   SYMLINKS
walkthrough  cursor  /tmp/ynh-walkthrough/project  N
```

### Clean up symlinks

```bash
walkthrough -v cursor --clean
```

Expected:

```
Cleaned cursor symlinks for persona "walkthrough" in /tmp/ynh-walkthrough/project
```

Verify:

```bash
ynh status
# Expected: No symlink installations found.
```

## 10. Prune orphaned installations

If a project directory is deleted while symlinks are still registered:

```bash
# Re-install symlinks
cd $YNH_WALKTHROUGH/project
walkthrough -v cursor --install

# Verify the installation is tracked
ynh status
```

Expected:

```
PERSONA      VENDOR  PROJECT                   SYMLINKS
walkthrough  cursor  /tmp/ynh-walkthrough/project  N
```

```bash
# Simulate orphan: delete the project
rm -rf $YNH_WALKTHROUGH/project

# Prune finds and cleans the orphan
ynh prune
```

Expected:

```
Removing orphaned installation: walkthrough (cursor) in /tmp/ynh-walkthrough/project
```

## 11. Install from Git URL

This is the primary real-world flow. Install a persona directly from a Git repo:

```bash
ynh install github.com/eyelock/ynh
```

This clones the repo, reads `.claude-plugin/plugin.json`, copies everything, and generates a launcher. The ynh repo itself is a persona with skills, agents, rules, and commands.

```bash
ynh ls
```

Expected:

```
NAME         VENDOR  INCLUDES   DELEGATES TO
walkthrough  claude  0 sources  0
ynh          claude  0 sources  0
```

Clean up:

```bash
ynh uninstall ynh
```

## 12. Pull skills from Git (with pick, path, and ref)

Create a persona that composes skills from external repos:

```bash
mkdir -p $YNH_WALKTHROUGH/composed-persona/.claude-plugin
cat > $YNH_WALKTHROUGH/composed-persona/.claude-plugin/plugin.json << 'EOF'
{
  "name": "composed",
  "version": "0.1.0"
}
EOF

cat > $YNH_WALKTHROUGH/composed-persona/metadata.json << 'EOF'
{
  "ynh": {
    "default_vendor": "claude",
    "includes": [
      {
        "git": "github.com/brianlovin/claude-config",
        "pick": ["skills/simplify"]
      },
      {
        "git": "github.com/eyelock/ynh",
        "ref": "main",
        "pick": ["skills/ynh-validate"]
      }
    ]
  }
}
EOF
```

This tests three include features:
- **pick** - only the named artifacts are included, not the whole repo
- **ref** - pins to a specific branch (also supports tags like `v1.0.0` and commit SHAs)
- **Multiple sources** - skills from different repos composed into one persona

Install:

```bash
ynh install $YNH_WALKTHROUGH/composed-persona
```

Run and check that only the picked skills are available:

```bash
composed "what skills do you have?"
```

The session should have `simplify` (from brianlovin's repo) and `ynh-validate` (from the ynh repo), but nothing else from either source.

## 14. Delegation

Create a team persona that delegates to another persona. The `delegates_to` URL must point to a real Git repo containing a persona - substitute your own below:

```bash
mkdir -p $YNH_WALKTHROUGH/team-persona/.claude-plugin
cat > $YNH_WALKTHROUGH/team-persona/.claude-plugin/plugin.json << 'EOF'
{
  "name": "team-test",
  "version": "0.1.0"
}
EOF

cat > $YNH_WALKTHROUGH/team-persona/metadata.json << 'EOF'
{
  "ynh": {
    "default_vendor": "claude",
    "delegates_to": [
      {"git": "github.com/<your-user>/<your-persona-repo>"}
    ]
  }
}
EOF
```

**Note:** Replace `github.com/<your-user>/<your-persona-repo>` with a real Git URL containing a persona. Delegation resolves the delegate at runtime (`ynh run`), so the repo must exist and be cloneable. Installation itself only copies local files.

Install:

```bash
ynh install $YNH_WALKTHROUGH/team-persona
ynh ls
```

Expected:

```
NAME         VENDOR  INCLUDES   DELEGATES TO
composed     claude  2 sources  0
team-test    claude  0 sources  1
walkthrough  claude  0 sources  0
```

When running `team-test`, ynh resolves the delegate persona, generates a vendor-native agent file, and the AI can delegate to it as a subagent.
