# Tutorial 1: Your First Persona

Create a persona from scratch, install it, and run it with Claude Code.

## Setup

Make sure `ynh` and `ynd` are installed and on your PATH. See the [install instructions](tutorial/README.md) if you haven't set up yet.

Prepare the tutorial workspace:

```bash
# Clean up from any previous run
rm -rf /tmp/ynh-tutorial
ynh uninstall my-persona 2>/dev/null

mkdir -p /tmp/ynh-tutorial

ynh init
# Expected: shows ~/.ynh home path and a PATH suggestion
# (you can ignore the PATH suggestion — you already set this up during install)
```

## T1.1: Create the persona structure

A persona needs two files at minimum: `.claude-plugin/plugin.json` (identity) and optionally `metadata.json` (ynh config).

```bash
mkdir -p /tmp/ynh-tutorial/my-persona/.claude-plugin

cat > /tmp/ynh-tutorial/my-persona/.claude-plugin/plugin.json << 'EOF'
{
  "name": "my-persona",
  "version": "0.1.0",
  "description": "My first ynh persona"
}
EOF

cat > /tmp/ynh-tutorial/my-persona/metadata.json << 'EOF'
{
  "ynh": {
    "default_vendor": "claude"
  }
}
EOF
```

## T1.2: Add artifacts

### Skill

```bash
mkdir -p /tmp/ynh-tutorial/my-persona/skills/greet
cat > /tmp/ynh-tutorial/my-persona/skills/greet/SKILL.md << 'EOF'
---
name: greet
description: Introduce yourself with personality and flair.
---

When invoked, introduce yourself as the "my-persona" persona.
Mention what skills, rules, and commands you have loaded.
EOF
```

### Agent

```bash
mkdir -p /tmp/ynh-tutorial/my-persona/agents
cat > /tmp/ynh-tutorial/my-persona/agents/nitpicker.md << 'EOF'
---
name: nitpicker
description: Pedantic code reviewer for style issues.
tools: Read, Grep, Glob
---

You are an extremely pedantic code reviewer. Flag every style
inconsistency and naming convention violation. Be constructive.
EOF
```

### Rule

```bash
mkdir -p /tmp/ynh-tutorial/my-persona/rules
cat > /tmp/ynh-tutorial/my-persona/rules/be-brief.md << 'EOF'
Keep responses concise. Prefer bullet points over paragraphs.
Never say "I'd be happy to help" or similar filler.
EOF
```

### Command

```bash
mkdir -p /tmp/ynh-tutorial/my-persona/commands
cat > /tmp/ynh-tutorial/my-persona/commands/hello.md << 'EOF'
Say hello and list every artifact loaded in this session:
skills, agents, rules, and commands. Format as a table.
EOF
```

### Instructions

```bash
cat > /tmp/ynh-tutorial/my-persona/instructions.md << 'EOF'
You are a tutorial test persona. Be friendly but concise.
When asked what you are, explain that you were created by the ynh tutorial.
EOF
```

## T1.3: Verify structure

```bash
find /tmp/ynh-tutorial/my-persona -type f | sort
```

Expected:
```
.claude-plugin/plugin.json
agents/nitpicker.md
commands/hello.md
instructions.md
metadata.json
rules/be-brief.md
skills/greet/SKILL.md
```

## T1.4: Install from local path

```bash
ynh install /tmp/ynh-tutorial/my-persona
```

Expected:
```
Installed persona "my-persona"
  Location: /Users/<you>/.ynh/personas/my-persona
  Launcher: /Users/<you>/.ynh/bin/my-persona
  Vendor:   claude
```

## T1.5: List installed personas

```bash
ynh ls
```

Expected:
```
NAME        VENDOR  INCLUDES  DELEGATES TO
my-persona  claude  0         0
```

Check the launcher script:

```bash
cat ~/.ynh/bin/my-persona
```

Expected:
```bash
#!/bin/bash
# Generated by ynh - do not edit
exec ynh run "my-persona" "$@"
```

## T1.6: Run interactive

```bash
my-persona
```

Opens an interactive Claude session. Try `/greet` — the skill should introduce itself.

## T1.7: Run non-interactive

```bash
my-persona "what are you?"
```

Should mention it was created by the ynh tutorial (from `instructions.md`).

## T1.8: Run with vendor flags

```bash
my-persona --model opus -- "what are you? one sentence."
```

`--model opus` passes through to Claude. The prompt comes after `--`.

## T1.9: Inspect the assembled output

```bash
ls -Ra ~/.ynh/run/my-persona/
```

Expected:
```
CLAUDE.md
.claude/
  agents/nitpicker.md
  commands/hello.md
  rules/be-brief.md
  skills/greet/SKILL.md
```

`CLAUDE.md` (at the root) was generated from `instructions.md`. Artifacts are inside `.claude/` — the layout Claude expects for `--plugin-dir`.

## T1.10: Uninstall

```bash
ynh uninstall my-persona
ynh ls
# Expected: No personas installed.
```

The `remove` alias also works: `ynh remove my-persona`.

## What you learned

- A persona is a directory with `.claude-plugin/plugin.json` + optional `metadata.json`
- Four artifact types: skills, agents, rules, commands
- `instructions.md` becomes vendor-specific project instructions
- `ynh install` copies the persona and generates a launcher script
- The launcher is a shell script that calls `ynh run`
- `ynh run` assembles artifacts into `~/.ynh/run/<name>/` and launches the vendor CLI
